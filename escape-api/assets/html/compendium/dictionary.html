Helpful Terms

Just a list of terms found elsewhere in the Compendium that may be confusing.

Noise

The primary way of "seeing" where other players are. When a player makes a Noise, a message is broadcasted to every player in the lobby that says something like "Player 'raklan' made noise at [A-1]!", where A-1 is the Sector on the map in which the noise is made. By carefully watching where other players are making Noise, you can deduce where they are and where they're going, but remember: a Noise COULD be where that player is, but it can also be a lie!

Sector

Synonymous with a space on the game board. The official rules use the term Sector, so I try to use it as well.

Stack (of a Status Effect)

Another way of saying a use, activation, or charge. For example, if you have 1 stack of the Adrenaline Surge Status Effect, you can receive the benefit of it 1 time before it goes away. If you have 5 stacks of it, you can receive the benefit 5 times before it goes away.

Game Configuration Options

Team Settings

Humans

The number of Human player slots to allow. There is no guarantee all of these spots will be filled, but instead allows that many Humans at most.

Aliens

The number of Alien player slots to allow. There is no guarantee all of these spots will be filled, but instead allows that many Aliens at most.

Manual Team Assignment

Each player in the lobby is listed in this section, as well as an option to randomly assign their team ("Default"), or to manually place that player on the Spectator, Human, or Alien team. As a note, players can individually choose whether to be a spectator during the pre-game lobby, but the settings defined here by the host will override that preference, unless "Default" is selected.

Roles

Using this section, you can set how many of each Role can and must be in the game. Each role has two number fields under it - Available and Required - which are explained below. For an explanation of each Role, see the Compendium page on Roles.

Available

Controls how many of a certain role is possible to be in the game, but not guaranteed. When roles are being assigned, the server picks roles for each player very similarly to a raffle drawing - it will create a pool of possible roles for each player, where every role receives a number of entries in the pool equal to the number entered in this field. It will then pick a random role from the pool for each player, removing the entry picked for that player from the pool of possible role assignments.

Required

Controls how many of a certain role the server should attempt to guarantee will appear in the game. This runs before the Available Roles process - when the server picks roles for each player, it will gather a list of all required roles from the Lobby Config. It will then go through that list one by one (in the same order the roles appear in the config window) and pick a random player on that team to assign the role to. Be aware that the server expects there to be enough players to fill the required roles set by the config. For example, if you have a total of 10 Human roles marked as required, but only 6 Human players, the game will not be able to start.

Cards

Working Pods

The number of Working Pod cards to add to the Escape Pod deck. These will be combined with Broken Pod cards into one deck. When a Human Player reaches any escape pod, they will draw a card at random from the Escape Pod Card Deck. If a Working Pod card is drawn, the pod is working and the player successfully escapes. If a Broken Pod card is drawn, the pod is not working, and that player must try to reach a different escape pod space. For more details, see How To Play - Escape Pods.

Broken Pods

The number of Broken Pod cards to add to the Escape Pod deck. These will be combined with Working Pod cards into one deck. When a Human Player reaches any escape pod, they will draw a card at random from the Escape Pod Card Deck. If a Working Pod card is drawn, the pod is working and the player successfully escapes. If a Broken Pod card is drawn, the pod is not working, and that player must try to reach a different escape pod space. For more details, see How To Play - Escape Pods.

Noise Cards

Using this section, you can set how many of each Noise card is added to the main deck. For an explanation of each card, see the Compendium page on Cards.

Item Cards

Using this section, you can set how many of each Noise card is added to the main deck. For an explanation of each card, see the Compendium page on Cards.

Modifiers

Various ways to modify your game can be found on this tab. An explanation of each option is below.

Turns

The number of Turns the game should last. By default, after this many turns have elapsed, the ship runs out of oxygen, and all remaining humans immediately die, ending the game. However, by turning on Survival Mode, any surviving humans will immediately and automatically escape after this many turns.

Auto Turn End

If enabled, the game will automatically end a player's turn after they move, if they do not have any cards in their hand. This is useful for those friends you have that are always forgetting to end their turn.

Game Modes

This section contains a collection of different game modes you can activate for your game, most of which are inspired by the original tabletop version's scenario tool. Each one has a unique effect on the game. Feel free to mix and match the different modes - you can activate any number of them at once. If two modes have a unique, conflicting, or non-obvious interaction, it will be noted in both Modes' descriptions.

  • Bloodlust Mode: When enabled, this changes the bonus given for killing players. Now, the Hyperphagic Status Effect gives a +1 movement bonus for EACH player killed, instead of a flat +1 bonus.
  • Destructive Attacks Mode: When enabled, this changes how Attacks are handled. Whenever a Sector is attacked, it degrades. Safe Sectors become Dangerous Sectors, and Dangerous Sectors become Wall Sectors. Any players that were in the Dangerous Sector that survived the attack before it became a Wall remain in that Sector and will have to leave it on their next turn.
  • Evacuation Mode: When enabled, this adds an evolution to the Game Board. At the beginning of a set turn, a random Dangerous or Safe Sector will be transformed into a special Evacuation Sector. Any human entering this new Sector immediately escapes, without needing to draw Escape Pod cards. The turn on which this sector should appear is set by a secondary control that appears in the Game Settings window upon activating this mode.
  • Infested Pods Mode: When enabled, this changes the starting location of Alien players. Aliens will now start in a randomly selected Escape Pod Sector, instead of an Alien Start Sector.
      Other Mode Interactions:
    • Scatter Mode: If both Infested Pods and Scatter are active, Infested Pods will override the start location for Alien Players, but still allow Human Players to be spawned randomly.
  • Last Man Standing Mode: When enabled, this changes the way Escape Pods are handled. At the beginning of the game, all Escape Pods will become Blocked, and cannot be entered. They remain Blocked until there is only one Human player remaining in the game, at which point they become Unblocked and may be used as normal.
      Other Mode Interactions:
    • Unstable Pods Mode: If both Unstable Pods and Last Man Standing are active, all Escape Pods are blocked until the unblocking criteria for both modes is met. For example, if both are active, and Unstable Pods is set to "Even Turns" then Escape Pods will become unblocked on every even-numbered turn once there is only one Human player remaining.
  • Last Resort Mode: When enabled, this changes the way the game handles running out of Escape Pod Sectors. When all Escape Pods Sectors have been marked as used, the game will pick a random Dangerous or Safe Sector and turn it into a special Evacuation Sector. Any human entering this new Sector immediately escapes, without needing to draw Escape Pod cards.
      Other Mode Interactions:
    • Reactor Mode: If both Last Resort and Reactor mode are enabled, humans will still need to reach the Evacuation Sector after activating all Reactor Nodes.
  • Necrophagia Mode: When enabled, this changes the bonus given for killing players. Now, every player receives 1 Stack of Hyperphagic for killing any other player, regardless of team.
  • Reactor Mode: When enabled, this switches the game's Escape Pod logic. Now, every Escape Pod sector is a Reactor Node, and Escape Pod cards are no longer used. Humans can reactivate the Reactor Nodes by entering that sector and are no longer removed from the game upon doing so. Once every Reactor Node has been activated, the ship's onboard security is reactivated, killing the Aliens. Every surviving Human immediately and automatically wins!
      Other Mode Interactions:
    • Last Resort Mode: If both Last Resort and Reactor mode are enabled, humans will still need to reach the Evacuation Sector after activating all Reactor Nodes.
  • Relentless Aliens Mode: When enabled, upon death, Aliens will respawn at the start of their next turn at a randomly selected Alien Start Sector. If this is disabled (which is the default value), an Alien Player that dies will be placed on the Spectator team for the rest of the game.
      Other Mode Interactions:
    • Infested Pods Mode: If both Infested Pods and Relentless Aliens modes are enabled, upon death, Alien players will respawn at a randomly selected Escape Pod Sector at the start of their next turn.
    • Scatter Mode: If both Scatter and Relentless Aliens modes are enabled, upon death, Alien players will respawn at a randomly selected Dangerous, Safe, Human Start, or Alien Start Sector at the start of their next turn.
  • Scatter Mode: When enabled this changes the starting location of ALL players. Every player will now start in a randomly selected Alien Start, Human Start, Dangerous, or Safe Sector, instead of their team's Start Sectors.
      Other Mode Interactions:
    • Infested Pods Mode: If both Infested Pods and Scatter are active, Infested Pods will override the start location for Alien Players, but still allow Human Players to be spawned randomly.
  • Survival Mode: When enabled, this switches the game's end condition. After the maximum number of turns have elapsed, reinforcements arrive and save the remaining humans. All remaining humans immediately and automatically escape.
  • Unconfirmed Kills Mode: When enabled, Player deaths and uses of Status Effects that prevent death (such as Armored or Cloned) are no longer announced in Game Event Alerts.
  • Unstable Pods Mode: When enabled, this changes the way Escape Pods are handled. Escape Pods will become Blocked, and cannot be entered. They become unblocked and may be used as normal using logic set by a secondary control that appears in the Game Settings window upon activating this mode, which has the following options:
    • Even Turns: Escape Pods are unblocked and usable only on even-numbered game turns. (2, 4, 6, etc.)
    • Odd Turns: Escape Pods are unblocked and usable only on odd-numbered game turns. (1, 3, 5, etc.)
    • On Turn #: Escape Pods become unblocked and permanently usable, starting at the beginning of the set game turn number.
    Escape Pods are still single-use, and permanently become used as normal.
      Other Mode Interactions:
    • Last Man Standing Mode: If both Unstable Pods and Last Man Standing are active, all Escape Pods are blocked until the unblocking criteria for both modes is met. For example, if both are active, and Unstable Pods is set to "Even Turns" then Escape Pods will become unblocked on every even-numbered turn once there is only one Human player remaining.
